hexo blog架设指南

安装hexo

$ npm install -g hexo-cli

生成Blog目录

#  安装 Hexo 完成后，请执行下列命令，Hexo 将会在指定文件夹中新建所需要的文件。
$ hexo init <folder>
$ cd <folder>
#安装项目所需依赖
$ npm install

此时Blog项目目录结构如下

# 新建完成后，项目内子文件目录如下：
.
├── _config.yml # 网站的配置信息，可以在此配置大部分的参数
├── package.json # 应用程序的信息。EJS, Stylus 和 Markdown renderer 已默认安装，可以自由移除
├── scaffolds # 模版文件夹。当新建文章时，Hexo 会根据 scaffold 来建立文件
├── source # 资源文件夹是存放用户资源的地方
|   ├── _drafts
|   └── _posts #通过 hexo new 命令生成的文档保存在此  
└── themes # 主题文件夹。Hexo 会根据主题来生成静态页面

创建文章

$ hexo new <artcle name>
or
$ hexo n <artcle name>

预览服务

$ hexo server
or
$ hexo s

生成静态文件

$ hexo generate
or
$ hexo g

部署Blog

$ hexo deploy
or 
$ hexo d

部署到Git

# 安装插件
$ npm install hexo-deployer-git --save

#编辑_config.yml
# Deployment
# Docs: https://hexo.io/docs/deployment.html
# 修改deploy字段
deploy:
  type: git # 部署方式为GIT
  repo: https://github.com/ephowe/ephowe.github.io.git # 仓库地址
  branch: master # 分支

#部署
$ hexo d
#输入GITHUB帐号密码
